# TSQ Training Data

TSQ v0.7 creates small deterministic JSONL datasets from eval tasks, verifier outcomes, repair traces, and example report metadata.

## Dataset Files

The default build writes:

- `data/generated/tsq_supervised_train.jsonl`
- `data/generated/tsq_supervised_eval.jsonl`
- `data/generated/tsq_repair_train.jsonl`
- `data/generated/tsq_preference_pairs.jsonl`

Build them with:

```bash
python -m tsq.cli build-dataset \
  --out-dir data/generated \
  --include-example-reports examples/reports
```

Validate one file with:

```bash
python -m tsq.cli validate-dataset \
  --path data/generated/tsq_supervised_train.jsonl
```

Summarize one file with:

```bash
python -m tsq.cli dataset-summary \
  --path data/generated/tsq_supervised_train.jsonl
```

## Schemas

- `task_name`
- `split`
- `prompt`
- `constraints`
- `input_text`
- `target_text`

- `id`
- `prompt`

- `chosen_output`

- `reason`
- `metadata`

## Example Types

Current example types include:

- `constraint_following`
- `number_preservation`
- `date_preservation`
- `forbidden_word_avoidance`
- `code_fence_closure`
- `repair_from_verifier_feedback`
- `receipt_aware_response`
- `dynamic_precision_explanation`

Each example carries metadata such as `source_task`, `repairable`, `verifier_expected`, `precision_hint`, `cost_model_used`, and `tsq_version`.

## Repair Examples

Repair examples are generated by running repairable fixtures through `RepairAwareMockRunner`. The builder captures:

- the cheap failed output
- verifier failures
- a repair prompt matching the Transformers backend repair mode
- the minimal repair target
- the final verifier-passing output

These examples teach the model to respond to verifier feedback, not just to the original user prompt.

## Preference Pairs

Preference pairs are generated for repaired examples:

- `rejected_output`: failed cheap output
- `chosen_output`: repaired verifier-passing output
- `reason`: verifier failure repaired into constraint satisfaction

This prepares a later DPO-style or preference fine-tuning wave without implementing DPO in v0.7.
